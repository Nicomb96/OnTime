Se presenta el proceso de normalización aplicado al modelo relacional de OnTime, garantizando integridad, reducción de redundancias y una estructura eficiente mediante el uso de formas normales hasta alcanzar la Tercera Forma Normal (3FN).

Este proceso permitió identificar y corregir posibles dependencias funcionales indebidas, separando correctamente las entidades y atributos para optimizar el diseño lógico de la base de datos. Como resultado, se mejora la consistencia de los datos, se facilita el mantenimiento del sistema y se prepara la estructura para futuras ampliaciones.
